# Docs Hygiene Workflow

Use this workflow when changing README, docs indexes, architecture docs, setup docs, decisions, or documentation links.

## Steps

1. Read `README.md`.
2. Read `docs/INDEX.md`.
3. Read `.codex/manifest.yaml`.
4. Read `.codex/ROUTING.md` when route behavior is affected.
5. Add or update `docs/INDEX.md` entries for important docs.
6. Keep task progress out of durable docs.
7. Prefer cross-links over duplicated long explanations.
8. Verify local links and manifest-listed paths.

## Output

- Docs changed.
- Index entries updated.
- Broken or suspicious links found.
- Validation commands run.
